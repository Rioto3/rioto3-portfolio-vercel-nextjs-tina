## はじめに

経済産業省が提供する**gBizINFO API**という存在を知ったものの、その有用性がわからないまま早一年が経過しました。

ところで最近、「ニュース砂漠」という言葉を知りました。インターネットの普及によってマスメディアの経営基盤が弱まり、新聞が消えた地域で汚職が増えるという現象が報告されています。

そこで私は**gBizINFO APIを使用することで、私のような一般人でも客観データから既得権益の傾向を見つけられるのでは**という仮説を立て、検証を試みました。

*参考：[朝日新聞「日本でもニュース砂漠？　ジャーナリズムの衰退がもたらすものとは」](https://www.asahi.com/articles/AST3F1TTNT3FUPQJ00HM.html)*

### 本研究の目的と概要

この調査は、「誰でも使えるツールで既得権益の構造を可視化できるのか？」という素朴な疑問から始まった最初の一歩です。完璧な調査というより、この可能性を模索する実験的取り組みとして捉えていただければ幸いです。

具体的には以下の3つの点を探ってみました：

1. **実験**: オープンアクセスなAPIを活用した公的資金監視の可能性
2. **試行**: 補助金の異常なパターンを統計的に検出する方法の模索
3. **着想**: 市民によるデータ駆動型監視の新しい形の提案

これらは完成された研究ではなく、むしろ「このアプローチが使えるかも」という可能性を示す第一歩です。この記事を叩き台として、より多くの方々がデータを活用した公的資金の監視に取り組み、より洗練された方法論や深い知見が生まれることを期待しています。

技術的な限界はあるものの、少なくとも「市民の手によるデータ監視」という新たな可能性の扉は開けたのではないかと考えています。

## データが語る不透明な公的資金の流れ

### 調査の出発点：二段階アプローチ

本調査は「既得権益をAPIで探知できるか」という仮説から出発し、**データによる公的資金監視の新たな可能性**を探求しました。二段階アプローチは以下のように構成されています：

1. **既知の不正事例の検証**：報道された不正受給事例がAPIデータに反映されているか
   * 信頼性確認のための予備調査
   * APIデータの限界を理解するための基盤作り
2. **未知のパターン発見**：データ分析から新たな既得権益の兆候を検出
   * **本研究の核心的価値**
   * 報道に依存しない市民主導の監視体制の可能性
   * 統計的異常値からの新たな不透明性の発見

### パート1: 既知の不正事例とAPIデータの整合性

今回は、私の住む宮城県の不正事例を調査対象に選びました。

客観的な調査を目指しつつも、身近なニュースから得られる実感を重要視したためです。特に東日本大震災の復興資金は現在、2021年度から2025年度までの5年間を**第2期復興・創生期間**という位置付けでもあり、既得権益や不正が生じやすい環境であると考えられます。

まず、報道された不正受給事例と経済産業省のウェブサイトで公開されているデータの整合性を検証しました：

#### カネヨ山野辺水産の事例

> 東日本大震災の復興支援補助金を不正受給したとして2025年3月に塩釜市の水産加工会社元社長が逮捕（[朝日新聞](https://www.asahi.com/articles/AST353QK9T35UNHB00LM.html)）

この事件は比較的新しく、まだ有罪確定はされていません。メディアは逮捕段階では報道するものの、判決段階では報道しない傾向があります。
そのため、データから容疑者の潔白を証明するツールとしての可能性も期待しました。

[経済産業省データベース](https://info.gbiz.go.jp/hojin/ichiran?hojinBango=6370601000166)では企業情報は存在するものの、補助金受給データが一切記録されていませんでした。

これでは、手法的にデータを取得できていない可能性があります。追加で検証してみます。

#### 検証結果

これらの検証から**公開データには重大な限界がある**ことが判明しました。
そもそも、今回合計５社を検索しましたが、ニュースで取り上げられた補助金に関連する情報が含まれていません。

サンプルサイズ５における調査ですが、
このデータベースが**既知の不正事例の検証**に使用できるほどの信頼性がないことはわかりました。

経済産業省が運営するこのデータベースは、各省庁から提供されるオープンデータを集約する設計ですが、実際には重要な補助金情報が欠落しています。
特に厚生労働省管轄の雇用調整助成金や復興支援関連の補助金が反映されていない点は、公的資金の透明性における構造的問題を示しています。

### パート2: 補助金受給パターンから見える異常値

上記検証にて、データベース自体信頼できないことがわかりましたが、流石にビックデータを謳っているプロジェクトですし、2020年においては110万件程度の更新があるものです。個別の木がなくても、植生のようなトレンドがあるかもしれません。

#### 統計的分析手法と発見

限界を認識しつつも、APIの強みを活かした分析を進めました。補助金受給回数の観点に基づいて企業を分類したところ、大きく次のグループに分かれることが判明しました：

1. 大企業・公益企業（受給）
2. **特異な民間企業**（受給）
3. 自治体・大学などの公的機関・（多数受給）

特に注目すべきは2つ目のグループです。通常、民間企業が補助金を受け取ること自体が珍しいにもかかわらず、突出して多くの補助金を受給している企業が存在します。

この異常検出には以下の手法を用いました：

* 補助金受給回数の度数分布分析
* 組織種別ごとの受給回数の箱ひげ図によるアウトライアー検出
* 時系列データにおける急激な変化ポイントの特定

これらの手法により、統計的に有意な「異常値」を持つ組織を特定することができました。

#### 注目すべき事例の詳細検証

組織種別と補助金受給回数の分析から抽出された異常値について、特に象徴的な２社について詳細な考察を行いました：

##### 一般社団法人環境共創イニシアチブの事例

* 補助金受給回数：45件（91位）
* 総額：約1兆572億円

この団体について調査したところ、朝日新聞（2020年9月24日）の報道で「電通が設立した法人であり、経産省の補助金事業の9割が競争なしで決まっていた」という事実が判明しました。[朝日新聞「電通設立の法人、競争なく9割受託　経産省の補助金事業」](https://www.asahi.com/articles/ASN9R6HP5N93UUPI001.html)によれば、公募に他の事業者が参加せず、さらに事業のほとんどが電通に再委託されていたという構造的問題が指摘されています。

APIデータから統計的に検出した異常値が、実際の問題（競争なしの受託、不透明な再委託構造）に関連していたことが確認できました。これは統計的アプローチの有効性を示す重要な発見です。

##### ランドブレイン株式会社の事例

* 補助金受給回数：19件（472位）
* 総額：約296億円

シンクタンクとして活動するこの企業について詳細調査を行ったところ、以下のような特徴が見つかりました：

1. ホームページ上での役員情報の明示的な記載なし
   ([ホームページURL](https://www.landbrains.co.jp/))
2. 日本民間公益活動連携機構（JANPIA）における役員名簿では、代表取締役吉武祐一氏以外が明示的に黒塗り
   ([資料PDF](https://www.janpia.or.jp/koubo/2020/download/syorui/normal/03/syorui20_03_7_2.pdf))
3. 2023年に宮崎県串間市発注の公共事業をめぐり、官製談合防止法違反で社員が逮捕される事案があった（後に処分保留で釈放されたが、詳細説明なし）
   ([弊社社員の逮捕について](https://www.landbrains.co.jp/hp/doc/231117.pdf))
   ([処分保留で釈放](https://www.landbrains.co.jp/hp/doc/231208.pdf))
4. 貸借対照表によれば資産合計約27.5億円、利益剰余金約17.8億円と公的資金への依存度が高い事業構造が推測される
   ([決算情報](https://catr.jp/companies/ab58b/109037/settlements/dbd84/299255))

一般的な株式会社としては極めて異例の情報公開姿勢であり、公的資金を多数受給している企業としての説明責任に疑問が残ります。

## 発見と考察

### データ監視の可能性と限界

この研究を通じて、データ監視の可能性と限界について以下のことが明らかになりました：

#### 可能性

* **パターン検出能力**: 統計的手法により、「通常とは異なる」補助金受給パターンを客観的に検出できる
* **費用対効果**: プロの調査報道と比較して極めて低コストで実施可能
* **広範な監視**: 地理的・時間的制約なく、全国規模の監視が可能

#### 限界

* **データの不完全性**: 省庁間で情報共有が不十分で、重要データの欠落が多い
* **因果関係の特定困難**: 「異常」が検出できても、その理由までは統計からはわからない
* **二次検証の必要性**: 異常検出後には、追加の調査・取材が不可欠

これらの発見は、データ監視を単独の手法ではなく、従来の調査報道の「補完ツール」として位置づける重要性を示唆しています。

### 市民ジャーナリズムの新たな形

本研究の結果は、データを活用した市民ジャーナリズムの可能性を示しています：

1. **民主化された監視機能**: 専門知識がなくても使える分析フレームワークの確立
2. **透明性向上への貢献**: 市民による監視圧力が行政の透明性向上を促進
3. **メディア弱体化の補完**: 「ニュース砂漠」では特に重要な、市民による監視機能の提供

特に注目すべきは、生成AIとの組み合わせによる効果です。データ分析結果をChatGPTやClaudeに入力することで、専門知識がなくても洞察を得られるようになり、市民と専門家の知識格差が縮小しつつあります。

### 公的資金の透明性確保に向けた課題

本研究から見えてきた公的資金の透明性に関する課題は以下の通りです：

1. **省庁間データ統合の不足**: 各省庁のデータが適切に統合されておらず、全体像が把握しにくい
2. **検索性・アクセス性の制限**: APIの仕様が複雑で一般市民には使いにくい
3. **情報公開制度の形骸化**: 形式的には情報公開されていても、実質的な監視機能を果たせていない

これらの課題に対処するには、技術的解決策だけでなく、制度的・文化的な変革も必要です。例えば、公的資金受給者の説明責任を強化する法整備や、より市民に使いやすいデータ提供システムの構築などが考えられます。

## 結論と今後の展望

### 研究成果のまとめ

**GBIZINFO APIを「既得権益の傾向調査」というテーマで活用できる一例を示すこと**ができました。
加えて、調査の過程において、このAPIが公衆による公的資金監視ツールとして有効に機能する可能性まで出てきました。

今回は、企業や団体に対する補助金の配分回数というパターンを統計的手法から鳥瞰することで、異常値が既得権益になっているのでは？という仮説の元で調査を行い、具体的にそのような兆候のある要素が確認できました。

このアプローチの最大の意義は、専門的な調査ジャーナリズムのリソースがなくても、公衆が低コストで既得権益を監視できる点にあります。
特に、生成AIツールの入力データセットとして使用可能であるという事実は、従来必須であった知識や論理的思考能力さえも公平になり、市井において不透明性について質の高い議論を提供し、民主主義を強固にします。

しかし同時に、このAPIには技術的限界（データの欠落、時間範囲）があることも確認されました。これらの制約は前提として使用すべきです。

あくまで今回は、GBIZINFO APIの活用を考えたいというモチベーションからスタートしたので、学術的観点から客観的に評価するという範囲で終了とします。経済産業省の本APIに関するQ＆Aにある「オープンデータとして公開可能なもの」を活用する範囲に留め、特定の組織や企業に対する真偽までは考察・保証を行いません。それはより適切な場で改めて検証されるべきです。

* [経済産業省HP よくあるご質問（FAQ）: https://info.gbiz.go.jp/faq/](https://info.gbiz.go.jp/faq/)

### 今後の調査方向性

今回は、2020年度の補助金受給回数という指標に着目しましたが、今後は以下のような多面的アプローチが考えられます：

1. **時系列分析の拡充**: 複数年度のデータを用いた長期的傾向分析
2. **空間分析の導入**: 地域ごとの受給パターン差異の検出
3. **ネットワーク分析**: 企業間・団体間の関係性から見る補助金の流れ
4. **テキスト分析**: 補助金名称や目的の言語分析による傾向把握

特に注目すべきは、自然言語処理技術と組み合わせることで、補助金名称や事業概要から「似たような補助金が異なる名前で繰り返し支給されている」といったパターンも検出できる可能性があります。

### 市民参加型監視の促進方法

この研究の成果を社会に還元するため、以下の取り組みを提案します：

1. **オープンソースツールの開発**: 誰でも使える分析ツールの公開
2. **ワークショップの開催**: データ分析の基礎を市民に教える場の提供
3. **定期レポートの公開**: 定点観測による継続的な監視と結果共有

この度使用したプログラムとデータセットはGithubに公開していますので、検証していただければ幸いです。

* [Githubレポジトリ: https://github.com/Rioto3/subsidy-analysis-jp/](https://github.com/Rioto3/subsidy-analysis-jp/)

### 生成AI活用のプロセス

本研究では以下の観点で生成AIツールを使用しました：

* Claude 3.7 Sonnet: コード作成・記事コンテンツの作成
* ニュース記事収集: ChatGPT・Perplexity

特に、データ分析の解釈段階では生成AIが大きな力を発揮しました。統計的な異常値を検出した後、その意味を理解するためにAIに質問することで、専門家でなくても深い考察が可能になりました。

## 実践ガイド：あなたもデータ監視を始めるには

### 初心者向け実践ステップ

gBizINFO APIを使った公的資金監視を自分でも始めたい方のために、ステップバイステップのガイドを提供します：

1. **準備段階**
   * APIキーの取得: [gBizINFO APIサイト](https://info.gbiz.go.jp/api/index.html)で無料登録
   * 基本的なPython環境の構築: Anacondaのインストールが最も簡単
   * Jupyterノートブックの設定: データ分析の標準ツール
2. **データ収集**
   * 基本的なAPI呼び出し方法
   * 大量データの効率的な取得テクニック
   * データの保存と管理
3. **分析の実施**
   * 基本統計量の計算方法
   * 異常値検出のシンプルな方法
   * 可視化テクニック
4. **結果の解釈**
   * 生成AIを活用した分析補助
   * 追加情報の調査方法
   * 結果共有の倫理とベストプラクティス

### 必要なスキルと環境

この取り組みを始めるのに必要なスキルレベルは、予想より低いです：

* **プログラミング**: 基本的なPython文法の理解（for文、if文程度）
* **統計**: 中央値、標準偏差など基礎的な概念の理解
* **環境**: ノートPCレベルの一般的な計算環境で十分

特に、コピー＆ペーストで使える分析スクリプトを用意していますので、プログラミング初心者でも取り組めるよう配慮しています。

### よくある課題と解決策

実践において遭遇しやすい課題と解決策を紹介します：

1. **API制限への対応**
   * 問題: リクエスト上限（1日10,000回）に達する
   * 解決: バッチ処理と時間間隔の設定
2. **データ欠落への対策**
   * 問題: 必要な情報が含まれていない
   * 解決: 複数ソースの組み合わせ（例: 法人番号APIとの連携）
3. **解釈の難しさ**
   * 問題: 統計的異常の意味理解が難しい
   * 解決: ChatGPTなどのAIを活用した解釈支援
4. **誤検出のリスク**
   * 問題: 統計的に異常でも実際は問題ない場合
   * 解決: 複数の指標での確認と外部情報との照合

法人の基本情報、財務情報、特許情報など以下のような様々なデータを取得できます。

#### ベース機能の実装コード

まず、gBizINFO APIと接続するための基本関数を示します。この関数は他の分析でも再利用可能です：

```python
import requests
import json
import pandas as pd
import time
from pprint import pprint

# APIの基本設定
BASE_URL = "https://info.gbiz.go.jp/hojin"
API_TOKEN = "YOUR_API_TOKEN"  # 実際のAPIトークンに置き換えてください
HEADERS = {"X-hojinInfo-api-token": API_TOKEN}

def search_company(company_name):
    """
    gBizINFO APIから企業情報を検索する関数
    """
    endpoint = f"{BASE_URL}/v1/hojin"
    params = {
        'name': company_name,
        'limit': '10'
    }
    
    try:
        response = requests.get(endpoint, headers=HEADERS, params=params)
        if response.status_code == 200:
            return response.json()
        else:
            print(f"エラー: HTTPステータスコード {response.status_code}")
            print(f"レスポンス: {response.text}")
            return None
    except Exception as e:
        print(f"例外が発生しました: {e}")
        return None

def get_company_subsidy(corporate_number):
    """
    法人番号を使用して補助金情報を取得する関数
    """
    endpoint = f"{BASE_URL}/v1/hojin/{corporate_number}"
    
    try:
        response = requests.get(endpoint, headers=HEADERS)
        if response.status_code == 200:
            return response.json()
        else:
            print(f"エラー: HTTPステータスコード {response.status_code}")
            print(f"レスポンス: {response.text}")
            return None
    except Exception as e:
        print(f"例外が発生しました: {e}")
        return None
```

#### 一般社団法人環境共創イニシアチブの分析コード

環境共創イニシアチブを対象とした分析コードは以下の通りです：

```python
def analyze_sii_subsidies():
    """
    一般社団法人環境共創イニシアチブ（SII）の補助金情報を分析する関数
    """
    # 法人番号で団体情報を取得
    corporate_number = "8010605002161"  # 環境共創イニシアチブの法人番号
    
    print(f"環境共創イニシアチブの補助金情報を取得中...")
    sii_data = get_company_subsidy(corporate_number)
    
    if not sii_data or "hojin-infos" not in sii_data or len(sii_data["hojin-infos"]) == 0:
        print("データの取得に失敗しました")
        return
    
    company_info = sii_data["hojin-infos"][0]
    
    # 基本情報の表示
    print(f"\n==== 一般社団法人環境共創イニシアチブ 基本情報 ====")
    print(f"法人名: {company_info.get('name', 'N/A')}")
    print(f"所在地: {company_info.get('location', 'N/A')}")
    print(f"法人番号: {company_info.get('corporate_number', 'N/A')}")
    print(f"設立: {company_info.get('date_of_establishment', 'N/A')}")
    
    # 補助金情報の取得と分析
    if "subsidy" not in company_info or len(company_info["subsidy"]) == 0:
        print("\n補助金情報はありません")
        return
    
    subsidies = company_info["subsidy"]
    print(f"\n==== 補助金情報 ({len(subsidies)}件) ====")
    
    # 補助金データをDataFrameに変換
    df = pd.DataFrame(subsidies)
    
    # 金額を数値型に変換（数値でないものはNaNに）
    df['amount'] = pd.to_numeric(df['amount'], errors='coerce')
    
    # 基本統計量の計算
    total_amount = df['amount'].sum()
    mean_amount = df['amount'].mean()
    max_amount = df['amount'].max()
    min_amount = df['amount'].min()
    
    # 府省別の集計
    dept_counts = df['government_departments'].value_counts()
    dept_amounts = df.groupby('government_departments')['amount'].sum()
    
    # 結果の表示
    print(f"\n● 補助金総額: {total_amount:,.0f}円")
    print(f"● 平均金額: {mean_amount:,.0f}円")
    print(f"● 最大金額: {max_amount:,.0f}円")
    print(f"● 最小金額: {min_amount:,.0f}円")
    
    print("\n● 府省別補助金件数:")
    for dept, count in dept_counts.items():
        print(f"  - {dept}: {count}件")
    
    print("\n● 府省別補助金総額:")
    for dept, amount in dept_amounts.items():
        print(f"  - {dept}: {amount:,.0f}円")
    
    # 特徴的な補助金の表示
    print("\n● 金額トップ5の補助金:")
    top_subsidies = df.nlargest(5, 'amount')
    for idx, row in top_subsidies.iterrows():
        print(f"  - {row['title']}: {row['amount']:,.0f}円 ({row['government_departments']})")
    
    # 時系列分析（認定日がある場合）
    if df['date_of_approval'].notna().any():
        df['date_of_approval'] = pd.to_datetime(df['date_of_approval'])
        df = df.sort_values('date_of_approval')
        
        print("\n● 時系列推移:")
        yearly_amounts = df.groupby(df['date_of_approval'].dt.year)['amount'].sum()
        for year, amount in yearly_amounts.items():
            print(f"  - {year}年: {amount:,.0f}円")

# 実行
if __name__ == "__main__":
    analyze_sii_subsidies()
```

def analyze_landbrains_subsidies():

        print(f"● 最大金額: {proc_max:,.0f}円")
        
        print("\n● 府省別調達件数:")
        for dept, count in proc_dept_counts.items():

        for dept, amount in proc_dept_amounts.items():
            print(f"  - {dept}: {amount:,.0f}円")

    if "subsidy" in company_info and len(company_info["subsidy"]) > 0:
        subsidies = company_info["subsidy"]
        print(f"\n==== 補助金情報 ({len(subsidies)}件) ====")
        
        # 補助金データをDataFrameに変換
        sub_df = pd.DataFrame(subsidies)
        
        # 金額を数値型に変換
